# Что делает проект?
Помогает быстро создать бота с цепочкой сообщений, которые отправляются через определенные промежутки времени. 
Пользователь подписывается на бота, оплачивает цифровой продукт и получает заранее подготовленный вами контент.

Платежи в этом боте принимаются в Telegram Stars, поскольку речь идет о цифровом продукте. Если вам нужно адаптировать
бота под продажу физических продуктов, подключите платежную систему в Telegram и [измените bot.send_invoice](timing/payments.py): 
укажите другие currency и provider_token. Если вы продаете цифровой продукт, менять ничего не нужно.
# Как работать с проектом?
## Создайте собственный файл .env в корневой папке
Заполните его, указав:

* TOKEN=токен вашего бота
* ADMINS=список id пользователей телеграм, которым будет открыт вход в админку
* HOST_REDIS=хост Redis, который вы используете
* PORT_REDIS=порт Redis
* USERNAME_REDIS=юзернейм Redis
* PASSWORD_REDIS=пароль Redis, который вы установили

## Измените файл config.py
В нем нужно указать в соответствующих строках название, описание, цену вашего продукта, телеграм-эффект при покупке,
а также список команд в меню бота. 

**Обратите внимание:** по правилам телеграм, команда paysupport должна быть обязательно!

**Важно:** бот настроен так, чтобы сообщения отправлялись каждый будний день. Если вы хотите это изменить, 
[откройте time_functions.py в папке timing](timing/time_functions.py) и измените строку 20.

## Измените файл messages.py в папке handlers
В нем вы увидите болванки - замените их на собственные тексты. В каждую функцию message_day_х можно добавить
отправку дополнительных сообщений (включая видео, фото и пр.) - в одну цепочку может входить столько сообщений, 
сколько вам нужно.

Если вы планируете оформлять возврат денег, раскомментируйте строку 18 (self.money_refund_message) и добавьте сообщение, 
а также напишите соответствующую функцию [в user_handlers.py в той же папке](handlers/user_handlers.py).

## Что делать, если нужно больше/меньше цепочек сообщений?
[В файле messages.py](handlers/messages.py) добавьте или уберите функции async def message_day_х. 
Затем исправьте функцию message_chain.
Всё остальное подтянется само.
